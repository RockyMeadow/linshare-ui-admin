Domains pattern

After creating a connection to an LDAP directory, it is necessary to create a domain pattern.

This pattern will allows you to define the correspondence between the firstname, lastname and email fields of {{'COMMON.NAME' | translate}}, and the attributes of your directory. {{ 'COMMON.NAME' | translate }} requires 4 different queries to work properly:

  • query LQL (Ldap Query Language) of authentication : query to find a user in the directory during the authentication phase from the identifier he has entered (email).
  • query LQL (Ldap Query Language) user : query to find a user in the directory when searching for consultation, editing, sharing, ...
  • query LQL (Ldap Query Language)autocomplete on all attributes: allows autocomplete a user from the entered pattern
  • query LQL (Ldap Query Language) autocomplete on firstname and lastname : allows autocomplete of a user from a couple of pattern: lastname-firstname or firstname-lastname
